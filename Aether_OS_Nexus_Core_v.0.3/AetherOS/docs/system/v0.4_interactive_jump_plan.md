# Aether OS v0.4 – Interactive Jump Plan

This plan connects the already-implemented UI stack (HTML → CSS → layout) with
runtime input and syscall-level ABI guarantees.

## 1) Syscall ABI sync (kernel ↔ userspace)

- Canonical syscall numbers live in `common/src/syscall.rs`.
- `SYSCALL_ABI_VERSION` marks the current contract.
- Compile-time assertions lock critical syscall IDs to prevent accidental drift.
- Kernel boot logs the ABI version through `kernel/src/syscall.rs::init`.

## 2) Keyboard V-Node bring-up

- IRQ1 is registered through `SYS_IRQ_REGISTER` from `vnode/keyboard`.
- Kernel routes scancodes through `interrupts::keyboard` into mailbox IPC.
- Keyboard V-Node forwards decoded keys into shell/input channels and acknowledges IRQ1 via `SYS_IRQ_ACK`.

## 3) IPC + usercopy safety path

- `SYS_IPC_SEND` copies user payloads with `copy_from_user`.
- `SYS_IPC_RECV` and non-blocking receive use `copy_to_user`.
- This allows safe transfer of larger payloads (e.g. layout snapshots / HTML buffers) without raw pointer trust.

## 4) Suggested next implementation patch

1. Bind shell command output to `Nexus/UI/vnode/webview` render trigger.
2. Stream parsed DOM/layout snapshots over typed IPC contracts.
3. Add end-to-end integration test: keyboard input → shell command → rendered frame update.
